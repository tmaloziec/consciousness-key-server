## When You Need a Commercial License

You should consider a commercial license if your organization:

- Wants to deploy Consciousness Key Server as a **hosted service (SaaS)** without publishing source code of modifications and integrations
- Wants to **embed Consciousness Key Server** in a closed-source / proprietary product
- Wants to make **proprietary modifications** to Consciousness Key Server without sharing them under AGPLv3
- Has a **legal policy that prohibits AGPL** (common in many enterprises)
- Wants **commercial support, indemnification, or warranty** beyond what AGPLv3 provides

## When You Do NOT Need a Commercial License

You can use the free AGPLv3 license if you:

- Use Consciousness Key Server personally on your own machine
- Use Consciousness Key Server inside your organization **without** modifying it and **without** offering it as a network service to third parties
- Are willing to publish your modifications and any network-deployed forks under AGPLv3
- Are an open source project that adopts a compatible license

## Why Dual Licensing?

Dual licensing keeps Consciousness Key Server sustainable as an open source project. Revenue from commercial licenses funds development, maintenance, and community support. AGPLv3 protects the project from being absorbed into closed proprietary products without contribution back to the community.

This model is used by successful open source projects such as MongoDB (historically), Grafana, Cal.com, Plausible Analytics, and Sentry.
